## 目的
- diopside（旧名称） におけるDB正本の保持・削除方針を定義し、配信データと運用証跡の判断軸を固定する。
- 削除/非公開化された動画を含む履歴情報をDBで追跡可能にし、再収集 と監査の再現性を維持する。

## 適用範囲
- 本書の対象はDB正本（`channels`/`videos`/`tags`/`video_tags`/`ingestion_*`/`recheck_*`/`publish_*`）のみとする。
- CloudWatch Logs、S3成果物、バックアップ世代、長期アーカイブの保持期間は本書の決定対象外とし、インフラ設計で管理する（旧参照、旧参照）。

## 保持対象（DB正本）
| データ区分 | 主な項目 | 保持目的 |
| --- | --- | --- |
| 動画・チャンネル正本 | `video_id`, `channel_id`, `title`, `published_at`, `channel_type` | 収集対象の正本管理と再配信再現 |
| タグ正本 | `tag_id`, `tag_type_id`, `is_active`, `updated_at` | タグ辞書の履歴一貫性維持 |
| 実行履歴 | `run_id`, `status`, `started_at`, `finished_at`, `failure_reason_code` | [[RQ-GL-002|収集実行]]・再確認・公開反映の監査 |
| 公開状態履歴 | `availability_status`, `availability_detected_at`, `availability_reason_code` | 削除/非公開検知後の除外根拠保持 |

## 保持期間（DB）
- DB正本データは原則無期限保持とする。
- 削除/非公開化された動画は物理削除せず、`availability_status` を更新したトゥームストーンとして無期限保持する。
- 物理削除を行うのは、法令・規約・セキュリティ事故対応で保持継続が不適切と判定された場合に限る。

## 削除条件
- 通常運用での削除/非公開検知は、配信対象除外（24時間以内）とDB状態更新で対応し、DB物理削除は行わない。
- 想定外のPII/秘密情報混入を検知した場合は、該当行を即時隔離し、当日中に物理削除または不可逆マスキングを実施する。
- 問い合わせ起点の削除要請は受付から3営業日以内に処理方針を回答し、実行結果を運用記録へ残す。

## 例外時の扱い
- 収集失敗や外部API制約で状態遷移を確定できない場合は、既存状態を維持し `availability_reason_code` に未確定理由を記録する。
- 誤検知により削除/非公開へ遷移した場合は、次回検証[[RQ-GL-002|run]]で状態を巻き戻し、巻き戻し理由を同一 `run_id` 系列に記録する。
- トゥームストーン情報に矛盾が検知された場合は、公開切替を停止し、整合復旧完了まで前回確定版を維持する。

## 監査可能性
- 公開状態遷移は最低限 `video_id`、状態、検知時刻、判定理由、起点[[RQ-GL-002|run]]を追跡可能にする。
- 監査時は「除外判定 -> DB状態更新 -> 配信除外」の順で証跡を辿れることを受入条件とする。
- 本書で規定しない非DB保持ポリシーは、インフラ文書側の証跡と突合して最終判定する。
