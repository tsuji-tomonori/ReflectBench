## スコープ注記
- 本文書は `docs/2.基本設計(BD)/03.アプリ(APP)` の旧文脈UI設計を保持する参考文書であり、現行スコープの正本ではない。
- 現行スコープの正本は `[[RQ-SC-001]]` と DD-INF/DD-APP 系列を優先する。

## 設計方針
- 本文書は、旧diopside文脈の画面を「利用者（旧定義）向け探索導線」と「[[RQ-SH-001|管理者]]向け運用導線」に分けて定義し、要求から設計への対応を明確にする。
- 画面ごとに目的・入口・主要操作・状態を定義し、利用者（旧定義）と [[RQ-SH-001|管理者]] が迷わず次操作へ進める情報設計を優先する。
- 画面一覧は見た目の列挙ではなく、旧利用シナリオを成立させる最小単位として管理する。

## 設計要点
- 利用者（旧定義）導線は「一覧表示（[[RQ-UC-002]]）→絞り込み/検索（[[RQ-UC-003]]/[[RQ-UC-004]]/[[RQ-UC-005]]）→詳細確認（旧UC群）」を基準とする。
- [[RQ-SH-001|管理者]]導線は「収集起動（[[RQ-UC-001]]）→監視/失敗調査（旧UC群）→再収集（旧UC群）→配信前後再確認/手動タグ付け（旧UC群）」を基準とする。
- 更新系UIはすべてBackend API経由でDB正本を更新し、利用者（旧定義）向け参照契約と直接結合しない。
- 詳細モーダル内の補助表示（コメント密度波形/ワードクラウド）は、主導線を阻害しない従属要素として配置する（旧ADR参照）。

## 更新系/参照系のUI境界
- 管理画面（UI-A01〜UI-A05）は更新系UIとしてBackend APIを必須経路にする。
- 利用者（旧定義）画面（UI-U01〜UI-U03）は参照系UIとして静的配信JSON/静的画像のみを参照する。
- 将来の高度検索は利用者（旧定義）画面に検索APIを追加可能とするが、管理画面導線は変更しない。

## UIコンポーネント基本設計（BD）
- 画面別コンポーネント設計は `[[BD-APP-UI-006]]` から `[[BD-APP-UI-011]]` を正本として管理する。
- UIプリミティブ設計（ボタン、チップ、スライダー、通知、入力）は `[[BD-APP-UI-012]]` から `[[BD-APP-UI-015]]` を正本として管理する。
- 色設計とトークン設計は `[[BD-APP-UI-016]]` を正本とし、`web/src/styles/global.css` の実装値との追跡性を維持する。
- 画面と部品の対応関係は `[[BD-APP-UI-005]]` のマトリクスで一元管理する。

## 補足
- 画面カタログは「どの画面が存在するか」だけでなく、「どの要求をどの操作で受けるか」を示す索引として扱う。
- 具体レイアウト、部品単位の表示仕様、例外メッセージ文言は [[BD-APP-UI-002]] と [[BD-APP-UI-004]]、および旧DD-APP-UI系列で詳細化する。
- 本書に記載の旧要求群/旧UC群/旧DD-APP-UI参照は、現行スコープでは正本トレース対象外とする。
